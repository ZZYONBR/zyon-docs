# Frota — Roteamento canônico de alertas (multas, IPVA, jornada)

**Decisão arquitetural** (gravada em 2026-05-05): a alocação real de placas a filiais é dinâmica e vem dos **manifestos ESL dos últimos 90 dias** (uso operacional efetivo), não do cadastro estático em `frt_veiculos_detran.filial_alocacao`.

## Hierarquia de resolução de filial

Para qualquer alerta operacional (multa, IPVA, licenciamento, etc.):

1. **Motorista identificado na multa** → usa filial do colaborador (`cs_colaboradores_cadastro.filial`)
2. **Sem motorista** → usa **`vw_frt_placa_filial_dominante`** (manifestos 90d, mais peso ao mais recente)
3. **Placa sem manifestos recentes** → fallback `frt_veiculos_detran.filial_alocacao`
4. **Sempre** dispara também alerta consolidado para **Jean Silva** (financeiro)

## Componentes vivos no Supabase

- `fn_frt_filial_codigo(p_filial)` — normaliza qualquer codinome ESL (`1.01 ICARO JOI`, `1.09 ICARO W-HOUSE`, etc.) para código curto canônico (`JOI`, `WHP`, `SAO`, `RJ`, `DEUTSCHE`, etc.)
- `vw_frt_placa_filial_dominante` — placa → filial canônica via manifestos 90d com `pct_filial_dominante` e `status_alocacao` (consistente / divergente / sem_cadastro)
- `frt_routing_rules` — regras canônicas (`tipo_evento`, `criterio_chave`, `criterio_valor`, `gestor_email`, `prioridade`)
- `fn_frt_route_alerta(tipo_evento, placa, filial_colaborador)` — retorna lista de gestores a notificar com prioridade

## Por que manifestos > cadastro DETRAN

Frota Ícaro é dinâmica: placas SXX0A* foram cadastradas como WHP mas hoje algumas estão em SP, outras em JOI temporariamente. Filial alocação cadastrada fica defasada.
