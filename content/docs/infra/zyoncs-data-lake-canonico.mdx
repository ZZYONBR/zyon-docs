### Fix SINISTROS (10/05/2026) — referência

**Caller**: `esl-pipeline` → `esl-ingest-template` → RPC `fn_ingest_sinistros` → tabela base `tracking.esl_sinistros` (view `public.cs_esl_sinistros` projeta).

**3 bugs sobrepostos identificados via análise direta do XLSX presigned do S3**:

- **Bug 1 (silencioso)**: ESL emite headers `Nota Fiscal/Número`, `Nota Fiscal/Série`, `Nota Fiscal/Volumes`, `Nota Fiscal/Valor NF` com F maiúsculo. `COLUMN_MAPS.SINISTROS` (v12) tinha apenas `Nota fiscal/...` com f minúsculo. Resultado: `nf_numero`, `nf_serie`, `volumes`, `valor_nf` ficavam NULL silenciosamente.

- **Bug 2 (fatal)**: ESL passou a emitir `RCTRC` (col 14), `RCTAC` (col 15), `RCFDC` (col 16) como categorias por modal (`Avaria`, `Extravio`, `Outros`), não numeric. RPC fazia cast direto `NULLIF(...)::NUMERIC` → erro `invalid input syntax for type numeric: "Outros"`. 18/18 jobs FAILED.

- **Bug 3 (perda de informação)**: tabela não tinha colunas para guardar a categoria do sinistro por modal — informação operacional valiosa para ATLAS/TORRE RA/ÍCARUS.

**Fix aplicado**:
1. Helper `public.fn_safe_numeric(text)` — cast tolerante (texto → NULL).
2. `tracking.esl_sinistros` ganhou 3 colunas TEXT: `rctrc_tipo`, `rctac_tipo`, `rcfdc_tipo` (final, preserva ordem da view).
3. View `public.cs_esl_sinistros` recriada com as 3 colunas anexadas no FINAL.
4. `fn_ingest_sinistros` reescrita usando `fn_safe_numeric` + extração da categoria via CASE quando valor não é numérico.
5. `esl-ingest-template` v13: COLUMN_MAPS.SINISTROS aceita ambos `Nota Fiscal/` e `Nota fiscal/` (retrocompat).

## Padrão canônico repo.py (federação Python)

```python
# zyon-agents/<agente>/repo.py
from supabase import create_client
import os

class Repo:
    def __init__(self):
        self._zyoncs = create_client(os.getenv("SUPABASE_URL"), os.getenv("SUPABASE_KEY"))
        self._rh = create_client(os.getenv("SUPABASE_RH_URL"), os.getenv("SUPABASE_RH_KEY")) if os.getenv("SUPABASE_RH_URL") else None
        self._lex = create_client(os.getenv("SUPABASE_LEX_URL"), os.getenv("SUPABASE_LEX_KEY")) if os.getenv("SUPABASE_LEX_URL") else None

    def get_zyoncs(self): return self._zyoncs
    def get_rh(self): return self._rh
    def get_lex(self): return self._lex

    def check_freshness(self, source_name: str) -> dict:
        """Gate obrigatório antes de tools L2/L3 críticas.

        Filtra por `source_name` (NÃO `source`). Retorna dict com chaves
        do schema PT-BR de vw_etl_freshness:
          source_name, entidade, origem, tabela_destino, fonte_canonica,
          is_critica, freq_esperada_min, cron_jobname, edge_function,
          ultima_atualizacao, minutos_atraso, status_freshness, observacoes.

        status_freshness ∈ {'OK','ATENCAO','CRITICO','SEM_DADOS','N/A'}.
        """
        return (
            self._zyoncs.from_("vw_etl_freshness")
            .select("*").eq("source_name", source_name)
            .single().execute().data
        )

    def is_fresh(self, source_name: str) -> bool:
        try:
            return self.check_freshness(source_name).get("status_freshness") == "OK"
        except Exception:
            return False
```

## Regras canônicas

1. **NUNCA** usar SQL cross-project (dblink, FDW). Sempre federação Python no agente.
2. **Sempre** consultar `vw_etl_freshness` antes de decisão crítica (tools L2/L3).
3. Agentes **leem** de `raw_*`/`mv_*`/`vw_*`/`public.*` em ZYONCS; **gravam** apenas em schema próprio (`atlas.*`, `hermes.*`, etc.).
4. Adicionar nova fonte → registrar linha em `etl_sources` ANTES de criar a ingestão.
5. Service-role key apenas no Vault Mac mini (DAEDALUS); agentes locais usam anon_key + RLS.
6. ZYON.RH e LEX são deliberadamente isolados — auditor externo (FINEP, advogado) acessa cada um sob protocolo próprio.
