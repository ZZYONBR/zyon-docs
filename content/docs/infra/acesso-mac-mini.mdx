## Modelo · 2 camadas

### Camada 1 · Tailscale SSH (identidade + rede)

- Identidade Tailscale (Google SSO via `@icaroexpress.com`)
- ACLs definem quem acessa qual host/porta
- SSH com certs auto-rotacionados a cada 12h para TI
- Logs centralizados no Tailscale Console

### Camada 2 · sudoers granular (comandos no Mac mini)

Hélio loga como user `rubens` mas só consegue rodar comandos pré-aprovados.

## Comandos para Hélio (referência rápida)

```bash
# Conectar via Tailscale SSH (uma vez por 12h é pedido confirmar identidade)
tailscale ssh rubens@mac-mini-de-rubens

# Ver agentes Python rodando
pm2 list

# Ver últimas linhas log do Cláudio
sudo pm2 logs claudio --lines 100

# Reiniciar Cláudio
sudo pm2 restart claudio

# Ver crons launchd do ZZYON
sudo launchctl list | grep zzyon

# Rodar FAT-JC importer agora
sudo launchctl start com.zzyon.atlas.fat-jc

# Ver últimas linhas log do FAT-JC
sudo tail -n 100 /Users/rubens/zyon-agents/logs/fat_jc.err.log

# Evolution status
sudo docker ps | grep evolution
sudo docker logs evolution --tail 50
```

## Onboarding de novo usuário TI

1. Adicionar email em `group:ti` do `tailscale-acl.json` (no zyon-agents)
2. PR no GitHub → Rubens aprova
3. Aplicar ACL no Tailscale Console
4. Convidar usuário pelo email no tailnet
5. Validar acesso

Nenhuma senha trocada, nenhuma chave SSH criada. Identidade unificada via Tailscale.

## Rescisão de acesso

Remover email de `group:ti` → propagação em menos de 1 minuto em todo tailnet. Sem precisar tocar no Mac mini.
