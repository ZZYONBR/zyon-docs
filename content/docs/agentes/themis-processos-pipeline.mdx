## Inferências automáticas do importer
- **tribunal** → do CNJ (TRT-12 do órgão 5 + tribunal 12, TJSC do órgão 8 + tribunal 24, etc.)
- **tipo** → órgão CNJ (5=trabalhista, 8=cível, 4=tributário/federal)
- **area_codigo** → órgão + heurística textual da classe
- **filial_codigo** → CNPJ Ícaro (mapeado em `lib/cnpjs_icaro.py`)

## Colunas obrigatórias
- `numero_cnj` (com ou sem máscara)
- `cnpj_icaro` (qualquer formato)

## Bug fix gravado
- `_to_float` no importer detecta automaticamente formato BR vs US:
  - "1.234,56" → 1234.56 (BR — vírgula é decimal)
  - "1234.56" → 1234.56 (US — ponto é decimal)
- Sem essa heurística, "180000.00" virava R$ 18.000.000,00.

**Why:** Rubens escolheu opção 1 (importar processos) como primeira frente após deploy. O caso Vanilson estava nas pastas físicas em `/Desktop/12. JURIDICO/Processos/` — usado como piloto live.

**How to apply:**
- Sempre validar com `--dry-run` antes de gravar.
- O CSV `processos_template.csv` é só o cabeçalho; criar uma cópia com nome significativo (`processos_2026.csv`, `processos_trabalhistas.csv`) para preservar o template limpo.
- Após import, rodar `sync_datajud` para puxar movimentações reais.
- DataJud só funciona se o CNJ realmente existe nos tribunais — processos administrativos (CARF, ANTT, ANVISA) não têm CNJ; usar `numero_administrativo` separado.
