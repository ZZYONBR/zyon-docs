## Limites conhecidos da Fase A (anotados para Fase B)

1. **Sem upload pro Storage** ainda — campo `arquivo_url` aceita link externo. Storage virá com Fase B.
2. **Sem IA preenchendo** — tudo é digitado manualmente. Fase B vai oferecer "Analisar com IA" que lê PDF da petição e sugere risco/fato gerador/docs/ações.
3. **Sem cruzamento operacional** — Fase C: para processo trabalhista de motorista, puxar diários de bordo + manifestos do período do ESL; pra avaria, puxar histórico do CTe. Schema já preparado (`fato_gerador_envolvidos JSONB`).
4. **Pessoas globais, não por processo** — quando cadastra pessoa nova no atribuir, ela vai pra `themis.pessoas` (compartilhada). Pode reaproveitar em outro processo.
5. **Histórico de análise** — `versao` incrementa, mas UI mostra só a mais recente. Fase B: tela de histórico mostrando evolução de risco ao longo do tempo.
6. **Validação por advogado** — campo `validado_por_advogado` está no schema mas UI ainda não tem botão "validar". Fase B: botão no painel + envio de notificação WhatsApp.
