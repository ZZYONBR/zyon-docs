## Schema do JSON que a IA retorna

System prompt da edge function impõe schema rígido:
```
risco, risco_justificativa, valor_condenacao_estimado,
valor_acordo_aceitavel, valor_provisao_recomendado,
fato_gerador_resumo, fato_gerador_data, fato_gerador_local,
fato_gerador_envolvidos, pedidos[], estrategia_defesa,
teses_defensivas[], documentos_sugeridos[], acoes_sugeridas[]
```

Pedidos têm tipo + valor + fundamento.
Teses têm tese + fundamento_legal (CLT art X / Súmula Y).
Documentos têm categoria + título + descrição + obrigatorio.
Ações têm título + descrição + tipo + prioridade + prazo_dias (offset).

## Bug fix 2026-05-08 — "Failed to fetch" no upload de PDF

**Sintoma**: ao clicar em "⚡ Analisar com IA" e anexar PDF, modal mostrava "Falha: Failed to fetch" antes do upload começar.

**Causa raiz**:
1. `<script type="module" src=".mjs">` carrega ES module mas **não expõe globals** — `window.pdfjsLib` permanecia undefined depois do `onload`.
2. Fallback usava `import('https://cdn.jsdelivr.net/...pdf.min.mjs')` — dynamic import cross-origin. Em produção Vercel, qualquer redirect ou header CORS faltante derruba com "Failed to fetch".

**Fix aplicado** (`torre-lex/src/lex-ia-analise.js`): trocar pdfjs-dist v4.7.76 (ESM-only) → **v3.11.174 (UMD legacy)** que expõe `window.pdfjsLib` como global ao carregar via `<script>` regular. Removido dynamic import. URLs CDN:
- `https://cdn.jsdelivr.net/npm/pdfjs-dist@3.11.174/build/pdf.min.js` (lib)
- `https://cdn.jsdelivr.net/npm/pdfjs-dist@3.11.174/build/pdf.worker.min.js` (worker)

**Smoke test em prod**: parsing de PDF mínimo retornou texto correto via preview MCP. Bundle pulou 281 KB → 292 KB.

**Como aplicar lição em outros torre-***: nunca usar dynamic `import()` de URL externa em código que vai pra Vercel. Sempre **`<script>` UMD** carregado on-demand via `loadScriptOnce()` helper.
