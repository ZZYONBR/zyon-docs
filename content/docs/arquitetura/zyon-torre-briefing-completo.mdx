### Orquestrador
**ZYON Orchestrator** — cérebro central. Correlaciona eventos cross-departamento (avarias SP + turnover SP → plano integrado). Princípio de **autonomia gradual**: começa em "monitorar e sugerir", ganha capacidade de ação direta conforme acurácia. **Ações > R$10K ou irreversíveis sempre passam por aprovação humana.**

### As 7 regras de ouro da separação dos agentes
1. Processos separados, servidor único (Docker Compose)
2. System prompts distintos por agente
3. Banco com schemas separados (PostgreSQL: schemas `roberval`, `iza`, `corporativo`, `shared`)
4. Credenciais isoladas (tokens OAuth, sessões WhatsApp, API keys nunca compartilhados)
5. Comunicação via Redis Pub/Sub (eventos)
6. Tools nunca compartilhadas
7. Logs centralizados mas segregados por namespace
