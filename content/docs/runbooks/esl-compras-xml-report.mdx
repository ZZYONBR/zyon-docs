## Restrição operacional descoberta

Na UI, só dá pra exportar XMLs de NFs **com NF Entrada manifestada** (ícone verde ✓ na coluna "NF Entrada"). NFs ainda pendentes de "Ciência da operação" / "Confirmação da operação" mostram checkbox cinza desabilitado e o botão Exportar retorna modal "Atenção: Selecione apenas notas que podem ser exportadas".

Implicação: o ingestor precisa rodar **manifesto SEFAZ DENTRO do ESL** antes de exportar, OU complementar com SEFAZ DFe direto via cert A1 PJ Ícaro.

## Tipo de XML retornado depende do estado da NF

| Estado UI | Coluna NF Entrada | Coluna Status | XML retornado |
|---|---|---|---|
| Manifestada + Confirmada | ✓ verde | ✓ verde | **`<nfeProc>`** completo (NFe + Signature XMLDSig + protNFe.infProt cStat=100) — XML SEFAZ AUTÊNTICO |
| Manifestada, não confirmada | ✓ verde | ⚠ amarelo | `<resNFe>` (resumo: chave + valor) — NÃO é XML autêntico, faltam dados |
| Não manifestada | ⓘ cinza | ⓘ cinza | **bloqueado** — modal "Selecione apenas notas que podem ser exportadas" (validação client-side) |

Validação no front roda em JS (não em backend). Para ter XML completo de uma NF, precisa primeiro **executar "Confirmação da operação"** (ação por linha ou em massa).

## Como obter ID interno da NF

Cada linha da tabela tem `<tr data-id="{internal_id}">` e o checkbox tem `value="{internal_id}"`. Não é a chave NFe (44 dígitos) nem o número da NF — é PK interna do ESL. Exemplo testado: NF número 2441 (MB FREIOS) → internal_id `489859`.

Para automação, recomenda-se cruzar com Data Export 6728 ou via list `/purchase/invoices?...` (HTML scraping ou JSON variant).

## Fluxo de auth para automação

ESL_TOKEN (Bearer) é só para `/api/*` (GraphQL + Data Export). Para rotas web (`/purchase/invoices/download_nfes`, `/user_notifications.json`) precisa cookie de sessão. Pattern para ingestor:

1. POST `/users/sign_in` com email+senha (Vault) → recebe `_eslcloud_session` cookie
2. GET `/purchase/invoices?...` para listar IDs internos (parsear HTML ou JSON variant)
3. POST `/purchase/invoices/download_nfes` com cookie + CSRF → enfileira
4. Polling `/user_notifications.json?per_page=10` → encontra nova `title="Notas de Compra"` com `link` S3
5. Download ZIP → unzip → `nfeProc`/`resNFe` por chave 44 díg
6. Persistir em `nfe_compras_xml` com `xml_tipo`
